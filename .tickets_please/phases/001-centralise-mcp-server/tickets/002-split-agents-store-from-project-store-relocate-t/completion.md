## Learnings
The Integrity variadic AgentStore parameter (rather than a required arg) was a deliberate backward-compat choice — many existing tests build a Store without a central data root, and forcing them all to wire one up was scope creep. The checkAgents bool guard prevents spurious "missing agent" warnings when no AgentStore is provided. WriteYAMLAtomic is sufficient for single-file agent writes; the StageOp tree (with .staging/ rename-into-place) is only worth it for multi-file atomic operations like ticket creation. Falling back DataRoot to DataDir+"-central" when DataRoot is empty (instead of erroring) keeps tests that build a Config{DataDir: t.TempDir()} working without modification — the alternative was forcing every test setup to mint a separate tempdir for agents. The cache's lookupAgentRef previously called c.store.ReadAgent; passing AgentStore into cache.New and routing through it is cleaner than going through the Service layer (avoids a circular dep). The flock helper in lock.go was kept on Store; AgentStore got a lightweight withGlobalLock that mirrors the same poll-loop pattern — duplication was minimal so extracting to a shared helper wasn't worth the indirection.
