Catch-all phase for defects found in tickets_please itself — web UI glitches, server bugs, MCP quirks. Filing bugs here keeps them separable from feature phases: feature work tracks scope, this phase tracks regressions and broken behaviour observed in real use (often spotted while using the remote instance against other projects like ensemble). Tickets here should describe observed vs expected behaviour and, where known, the page/component involved. No waves needed unless a cluster of related fixes emerges.
