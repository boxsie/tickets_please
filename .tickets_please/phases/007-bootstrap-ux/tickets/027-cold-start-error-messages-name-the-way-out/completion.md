## Learnings
The cold-start "no agent registered" error text wasn't asserted anywhere in mcptools tests (verified by grep) — the only similar svc-layer string lives at internal/svc/service.go:420 ("project %q not mounted") and is a separate code path. handleListProjects is the cheapest handler to exercise the no-session callWithRetry path in a test: it takes no required args and goes through callWithRetry directly. mcp-go's CallToolRequest{} zero-value (with Params.Arguments nil) is enough for callers that take no required args. freshToolsForRegister already wires a fakeEmbed-backed svc.Service plus an empty Registry; reusing it instead of building a custom fixture keeps mcptools tests consistent. Error message style across mcptools is "lowercase prefix: detail" (e.g. unauthenticated:, invalid argument:); the new bootstrap-guidance text breaks the line into a leading sentence + period-separated branches and stays under one logical paragraph; errors.Is(err, domain.ErrUnauthenticated) still matches because the wrap verb is %w. instructions_test.go snapshots load-bearing phrases AND enforces a length budget (currently [1000, 5000] chars) — adding a section to ServerInstructions needs to keep within the budget; sibling T2 should plan for this.
