## Learnings
tickets_please init ignores TICKETS_PLEASE_DATA_DIR / TICKETS_PLEASE_DATA_ROOT env vars when ~/.tickets_please/config.yaml is present — it reads data_dir/data_root from config and resolves them relative to the binary's cwd, so smoke-testing init in a tempdir requires either nuking the config (heavy hammer) or just inspecting whatever path the log line reports. The README lands at <data_dir>/README.md, not <repo>/.tickets_please/README.md, when data_dir is a relative path. dataDirReadme is a Go string-concat const (not a backtick raw string); changing the style would invalidate any blame tooling but is otherwise free. No tests snapshot or length-check this README, so it's safe to grow without test churn — but that also means no guard against accidentally dropping cold-start guidance later.
