## Learnings
instructions_test.go has two distinct guarantees that future edits to ServerInstructions need to satisfy: (1) the load-bearing phrase list (TestServerInstructionsContainsLoadBearingPhrases) — extend this when adding any new section that should never silently regress; (2) the [1000, 5000] character budget (TestServerInstructionsHasReasonableLength) — the budget has comfortable headroom (was 2864, ceiling 5000), so a single section addition is safe but the budget exists to stop the file from drifting into "everything dump" territory. The const uses Go string-concat (`+ "`...`" +`) to embed backticks for code spans inside a backtick raw string; preserve that exact style when editing — switching to escaped backticks or text/template would invalidate the prose-style tests. The "Bootstrapping" section deliberately echoes the same user-visible error string ("no .tickets_please/project.yaml at <path>") so an agent that hits the error mid-task can pattern-match it back to the persistent-context guidance — the two channels reinforce instead of saying different things.</learnings>
