## Learnings
When deleting a feature, the cheapest dead-code-free path is route then handler then page template then sidebar link then header button then CSS block then safelist entries then tests then e2e steps, in that order. Catching the dead waveSection type that survived the deletion (still referenced by phaseWithWaves) was a near-miss — go build with the unused-import error caught the leftover fmt import but a type that is referenced by another type would not get flagged at all; would have shipped silently if it were genuinely orphaned. CSS Grid with grid-template-columns auto-1fr-auto for the summary row is the right primitive when you want chevron plus flexible-middle plus fixed-right; flex with margin-left auto works too but grid handles wrapping more predictably. Status-distribution mini-bars are most legible at about 0.45rem height by 8rem width with no internal labels; the title attribute serves as the per-segment count tooltip. tabular-nums on the counts column keeps numbers vertically aligned across phase rows even with single-vs-double-digit values. The percentOf template helper handles total equal zero by returning 0 (avoids div-by-zero); empty-distribution rows render a faded base-color bar so the layout does not collapse on a phase with zero tickets. Pattern-matching tailwind safelist entries (regex on phase-row-bar-{column}) keep the safelist short when class names are server-rendered from enum values. Polish work that adds about 200 LOC of CSS while removing about 500 LOC of dead matrix code is a net simplification — both directions matter.
