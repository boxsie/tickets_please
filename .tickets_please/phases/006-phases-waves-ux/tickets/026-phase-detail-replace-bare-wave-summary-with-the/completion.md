## Learnings
Reusing a template fragment across pages is mostly a data-shape problem: changing phaseDetailData.Waves from a domain summary slice to the local waveSection slice was the only Go-side work because the template just consumes the iteration shape (Wave, Tickets, IsUnassigned). bucketTicketsByWave with a phase-scoped ListTickets call gives identical buckets to the phases-index path, so the visual output is guaranteed to match without copying logic. The phase-detail page did not have the index tinted body container so the wave block needed its own card-shaped wrapper (.phase-detail-waves) — pattern is block-level component class rather than context-dependent variant. Eliminating the ListWaves call here also makes the SPEC quirk (ListWaves with non-nil phase scopes correctly to that phase while nil-phase scopes to phase-less only) one less footgun the code has to know about — the page just walks the project tickets and buckets.
