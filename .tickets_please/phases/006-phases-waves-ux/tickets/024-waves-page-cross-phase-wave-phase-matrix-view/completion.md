## Learnings
Bucketing tickets into a 2D matrix with deterministic ordering is cleanest as a two-pass walk: first map[(wave,col)]*cell with placement counts, then materialise rows in wave-order with len(phases)+1 cells each. Trying to grow rows on the fly while iterating tickets makes wave 0 (sorted last) hard to handle. Treating orphan PhaseIDs as "place into Unphased column" keeps the hard-rule "never silently drop" satisfied without needing a banner for the common case — the banner only fires when cell counts genuinely sum to less than len(tickets), which would mean a programmer error in the bucketer rather than dirty data. CSS overflow-x:auto on a wrapper around a tight border-spacing:4px table gives a clean horizontal-scroll experience for projects with many phases (Liquidity HUD has ~12); the alternative of width:max-content forced layout shifts. The dot-distribution inside each cell is small enough (~0.45rem) that empty status counts can be omitted entirely without losing legibility — the title attribute on each dot doubles as the per-status count tooltip. The wave-expanded details element default-opens when the matrix is skipped (zero phases) which keeps the v1 flat-list shape intact for the no-phases case.
