## Learnings
A display:grid summary inside a details element treats every direct child including ::before pseudo-elements as a grid item, so a 4-column template with a chevron ::before quietly becomes 5 cells and wraps the description onto a second line. Switching to flex with an explicit span.phase-row-chevron is simpler and predictable. Tailwind purge runs from internal/web/static/_src/tailwind.config.js against ../templates/**/*.tmpl — adding new server-rendered classes (no literal in any template) requires both a safelist entry AND an npx tailwindcss rebuild + go build so the embedded CSS in the binary picks them up; restarting the systemd service alone is not enough since the binary embeds templates and CSS via go:embed. Reusing waveSection across two handlers required moving the type declaration to a section visible to both, but bucketTicketsByPhaseAndWave plus bucketTicketsByWave compose cleanly: the per-phase variant indexes tickets by PhaseID first then delegates to the per-wave bucketer, which keeps the wave 0 (unassigned) sorted-last convention in one place. Tests asserting "no cross-phase leakage" are easier to write structurally (split on the closing details tag, check each chunk) than via DOM parsing — keeps the dependency surface to plain strings.
