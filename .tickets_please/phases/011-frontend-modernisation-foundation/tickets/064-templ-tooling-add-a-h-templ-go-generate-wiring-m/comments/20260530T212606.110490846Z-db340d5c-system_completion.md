Learnings:
- **Don't `go run github.com/a-h/templ/cmd/templ` from a Makefile.** It transitively pulls cmd/templ's entire toolchain (`fatih/color`, `cli/browser`, `cenkalti/backoff/v4`, `x/tools/go/packages`, etc.) into the project's go.sum the first time you build. Use the installed binary via `command -v templ` and print an install hint if missing — keeps go.sum to the runtime dep only (`github.com/a-h/templ` for `templruntime`).
- **`go get <pkg>` to add a dep silently upgrades transitives.** `go get github.com/a-h/templ` bumped `x/crypto`, `x/net`, `x/text` and required `go mod tidy` to repopulate go.sum for unrelated go-git transitives (`x/net/proxy`, `x/net/context`). Always run `go mod tidy` immediately after `go get` and `go build ./...` before declaring victory.
- **templ generated files inherit the package of the source.** `hello.templ` has `package components` at the top; the generated `hello_templ.go` matches. The directive `//go:generate templ generate` in a sibling `.go` file is enough — no per-file directive needed.
- **Generated-files-checked-in is the right default for single-binary projects.** Otherwise a fresh clone needs `go install templ` before `go build` works, which contradicts the "single binary, single toolchain" ethos in SPEC.md. CI freshness is enforced by `make build` depending on `make generate` — drift fails the build because `git diff --exit-code` would catch a stale committed `_templ.go` once we add that to CI (out of scope here).
- **Datastar / Tailwind tickets in this wave will reuse the `generate` umbrella target.** When Tailwind lands its `make css` step, the convention is: add `.PHONY: css`, append `css` to the `generate:` dependency line. No further surgery in `build:` needed.
- **Dev-route gating is one-liner.** `if deps.Dev { mux.Handle(...) }` inside `Mount` is the cleanest spot — keeps the conditional out of the handler, and templ-hello/components-playground/sse-ping all sit naturally under `/_dev/*`.
- **Route registration in `Mount` runs before any request**, so the `if deps.Dev` branch decision is static-per-process. Confirmed by the smoke test: a server without `--dev` returns 404 even though the handler file is compiled in. No tree-shaking needed.
