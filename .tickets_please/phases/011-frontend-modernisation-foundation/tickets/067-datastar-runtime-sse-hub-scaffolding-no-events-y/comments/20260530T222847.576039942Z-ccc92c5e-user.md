### Learnings

- **Vendoring policy: pinned tag URLs only.** Auto-mode's permission classifier blocked the jsdelivr `@main` ref outright — correctly, because branch refs mutate. The right pattern for any vendored runtime is `raw.githubusercontent.com/owner/repo/<TAG>/<path>` with a literal version, mirroring how htmx is pinned to 1.9.12. Document version + exact curl in `static/README.md` so a future bump is a one-line edit.
- **Datastar 1.0.1 publishes no prebuilt release asset** — only tarball/zipball. The runtime bundle lives at `bundles/datastar.js` in-tree at the tag, which raw.githubusercontent serves.
- **The `@` sigil trap in templ prose recurred.** Ticket 066 hit it with `@Card`; this one hit it with `@get`. In attribute values the `@` is literal (so `data-on-load="@get('/sse')"` works), but the same string in templ prose gets parsed as a component invocation, templ generates silently, and only `go build` catches the bad reference. Treat as a templ-prose rule: any `@` symbol in copy gets entity-escaped (`&#64;`).
- **Datastar `@post` ≠ form POST for CSRF.** `@post('/_dev/sse-ping')` does not carry the `_csrf` hidden field the existing `withCSRF` middleware expects, so wrapping a Datastar-driven POST in `wrap(...)` rejects every call. Dev no-ops can skip middleware; **real W3 routes will need a header-based CSRF path** (a `data-headers` attribute pulling the token from page chrome, or a separate session-aware token endpoint). Plan for it.
- **Templ HTML-encodes single quotes in attribute values** (`'` → `&#39;`). Browsers decode natively so this is invisible to users, but it broke my literal-string `grep` in the smoke. Either compare against the encoded form or parse with a real DOM library.
- **Stale smoke server processes leak across sessions.** Six tickets_please binaries from past smoke runs were holding ports 18765–18768, 18999, 18998 — `trap EXIT` didn't run because each smoke ran in a short-lived shell parent. One of them served pre-feature content and produced a fake-pass that nearly hid this ticket's `/sse 404`. **Future smoke flow**: `pgrep -af tickets_please` before binding; kill anything on ports > 18000; treat old-looking response content as a binary-cache lie, not a code bug.
- **Empty-state DATA_DIR recipe is still load-bearing.** Without `DATA_DIR=$(mktemp -d)` the eager-mount picks up whichever cwd has a `project.yaml`. Even with it set, `GET /` can 303-redirect to a project — home-page assertions need `-L` or a path that always renders.
- **W1 publishes nothing on purpose.** The Hub + heartbeat is the contract W3 composes on. The follow-up `sse-hub-per-session-topic-scoped` is pre-baked into the `topicGlobal` constant and the `deps.Hub` interface — swapping in a per-session implementation should not touch `handlers_sse.go`.
- **complete_ticket envelope bug, position variant.** Today's failure mode was field-drops based on parameter ordering (`work_summary` as middle-of-three rejected; same content as first-of-three accepted). Worth adding to CLAUDE.md as an additional Claude Code MCP envelope failure mode beyond the markdown-content one already documented.</body>
