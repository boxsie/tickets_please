## Learnings
templ identifier collision: `templ Foo(...)` becomes a Go func, so a sibling `type Foo struct` is a duplicate decl. Hit this writing `Flash(f *Flash)`; renamed to `FlashBanner`. Avoid `web → components → web` import cycles by mirroring chrome types (Projects, AgentLabel, Flash, CSRF, ShowLocalBanner, URL) in the layout package as a local struct, converted once at the render boundary (`chromeToLayout`). `templ.WithChildren(ctx, page)` is the clean way to slot a page into a layout — the layout's `{ children... }` pulls from context. Inline `<script>` for the project-picker survives the port; tiny, self-bootstrapping via `dataset.pickerWired`. Generated `_templ.go` files are checked in — always `make generate` then stage both source + generated in the same commit. MCP comment commits sweep up dirty working-tree files: the templ work landed under a commit titled "comment on tickets-please/115" because an add_comment ran with the templ changes uncommitted. Lesson: commit your feat commit BEFORE calling any MCP tool that touches git. Subagent scope creep: the delegated agent also finished an unrelated in-progress ticket (115) it noticed in working state. Be explicit about scope. The `serve` subcommand uses `--addr 127.0.0.1:PORT`, not `--port`. Page() and RenderTempl() coexist on Renderer for W1 — old html/template Render stays alive until cleanup.
