## Learnings
- `internal/store/records.go` already imports `tickets_please/internal/domain` (TicketRecord uses `domain.Column`), so referencing `domain.Role` from the new MembershipRecord didn't introduce a new direction. The pattern is: store records own yaml tags, domain types are pure Go shapes, and the records package depends on domain (not the other way). Don't try to put the Role constants on the record side â€” they're a domain concern.
- The AgentStore's `withGlobalLock` uses `<Root>/.lock` as a constant filename. Pointing both AgentStore and UserStore at the same DataRoot makes them share that single lock file â€” correct for serialisation but chunkier than needed. Could split per-store later (`<Root>/.lock-users` etc.) if contention ever shows; not worth pre-optimising at homelab scale.
- `FindUserByOAuthSubject(provider, sub)` is a O(N) walk every call. Fine for a homelab (â‰¤dozens of users) but not free if the user count grows; the natural index is per-provider login â†’ user-id map. Out of scope here, but if W2-2 (OAuth callback) gets hot, that's the seam.
- The ticket's `GitHubLogin` field is a username string, not the stable numeric GitHub user-id. That's mildly fragile (GitHub allows renames), but matches the ticket spec exactly. If a future ticket needs rename-resilience, switch to storing `gh_user_id` and treat login as a display-only field. The OAuth provider ticket (86c81164 / W2-2) is the right place to make that call.
- `GrantMembership` is idempotent by *role*: same (user, project, role) is a no-op preserving the original GrantedAt; same (user, project) with different role overwrites and bumps GrantedAt. That keeps "grant viewer twice" from polluting audit history while still recording legitimate role changes. The test asserts both branches.
- `MembershipStore.GrantMembership` does its existence check inside `withGlobalLock` before write to dodge TOCTOU between two concurrent grants. Same pattern as AgentStore.RegisterAgent's active-key uniqueness check.
- The per-project subdirectory layout (`memberships/<project_id>/<user_id>.yaml`) was a deliberate choice from the ticket spec â€” `os.RemoveAll(membershipsDir/<project_id>)` is the entire DeleteProject cleanup, no walk-and-filter. Future ticket wiring DeleteProject just needs that one call.
- An empty subject in `FindUserByOAuthSubject` is rejected with a clear error rather than silently matching the first user with `*field == ""`. Defensive against an OAuth callback that fails to extract the subject claim.
- Test ergonomics: yaml time.Time round-trips lose sub-second precision, so the round-trip test rounds to `time.Second` before comparing. Otherwise the assertion fails on monotonic-clock components stripped by the marshal step.
- LSP "use of internal package not allowed" diagnostics fire because the workspace mismatch warning treats `/mnt/data/projects/tickets_please` (current cwd) and `/home/dan/Documents/projects/tickets_please` (the bind-mount source seen by gopls) as different modules. They're the same inode; ignore the noise and trust `go build ./...` / `make check`.
