## Learnings
Scoping a multi-layer ticket bottom-up (domain → store → cache → svc → mcp → web) with a build after each layer kept the rippling signature changes tractable. Two signatures changed: svc.RegisterAgent (+actingForUserID) and cache.New (+*store.UserStore).

Key gotchas for future-me:
- cache.New has exactly ONE production caller (svc/service.go) but NewWithStore has 10 test callers. Adding the UserStore param to New only — and having NewWithStore pass nil — kept all 10 test sites untouched. When extending a constructor, check whether a thin wrapper absorbs the test blast radius before editing every call site.
- DisplayName for ActingFor/created_for/etc. can't be resolved in record.ToDomain (no store handle). Two hydration sites: the cache (lookupUserRef, for reads — mirrors lookupAgentRef and degrades to id-only when UserStore is nil or the user file is missing) and the svc boundary (hydrateActingFor on the live agent, called in requireSession/optionalSession/GetAgent). Persist only the id; enrich the name where a store is reachable.
- The membership-enforcement chokepoint is `authorizeActingFor(agent, projectID, write)` called right after each mutation resolves its LoadedProject (lp.Project.ID), BEFORE taking lp.Lock. For a key-only agent (ActingFor==nil) it's a no-op, so existing behaviour is exactly preserved — that's what makes "agent without acting_for behaves as today" hold. Wired into all 8 mutation methods; a viewer role is rejected on writes (ErrForbidden).
- The RegisterAgent signature change is the kind that ripples to ~35 sites. A paren-aware python script that appends `, ""` only to 5-arg svc-style calls (skipping the 2-arg AgentStore.RegisterAgent and the handleRegisterAgent MCP handler, distinguished by top-level comma count) handled the multi-line test calls cleanly — a line-based sed would have missed the wrapped ones.
- DELIBERATE DEFERRAL: the acceptance asked for clickable /agents/{id} and /u/{user} links in the comment attribution. Those routes don't exist yet (they're Phase-2 W3 deliverables — agents-index-page / a user page), so I render "Claude (for Dan)" as plain text rather than anchors to 404s. The structured data (Comment.AuthorFor, Ticket.CreatedFor/CompletedFor) is fully persisted + hydrated, so wiring the links later is pure presentation. Worth a follow-up ticket once those pages land.
- TOOLING TRAP: `gofmt -w internal/ cmd/` reformats EVERY file with pre-existing gofmt drift, not just the ones you edited — it pulled ~12 unrelated files (vecindex/index.go, store/watch.go, etc., all comment-alignment churn) into the diff. Format only the files you touched (`gofmt -w <specific files>`), or the feature commit gets polluted. I staged only the feature files explicitly; the unrelated gofmt churn was left unstaged.
