- Hardcode the oauth2.Endpoint literals; do NOT import golang.org/x/oauth2/google — it drags in cloud.google.com/go/compute/metadata. Copying the two AuthURL/TokenURL pairs keeps go.sum to just golang.org/x/oauth2 (v0.36.0 via tidy). Verified no cloud.google transitive landed.
- The templ layout package has no standalone Base wrapper — only layout.Layout(PageData), which renders the FULL chrome (sidebar/nav). A pre-auth login page shouldn't show that, so login.templ emits its own minimal <html> and is served via Renderer.RenderTemplPartial (NOT RenderTempl, which double-wraps in Layout). Reuse existing CSS classes (empty-state, subtitle, page-actions, btn btn-primary) so it renders without a tailwind rebuild.
- API signatures bite: Renderer.RenderTemplError(w, r, status, err) takes an error (wrap strings in errors.New); RenderTempl(w, r, opts PageOpts, page) takes PageOpts — there is no status arg.
- UserStore had to be wired into svc.Service (074 only built the store type). Add the field to the Service struct AND build+assign it in the constructor's &Service{} literal next to AgentStore: as (easy to add the field but forget the construction → silently nil). Mirror store.NewAgentStore(dataRoot,…) with NewUserStore(dataRoot,…).
- Subjects match 074's store: github = login (+ /user/emails fallback for private email), google = OIDC sub. upsertUser does find-then-write inside UserStore.WithGlobalLock so concurrent first-logins can't dup a user (FindUserByOAuthSubject/WriteUser don't self-lock).
- Two cookies coexist by design: tp_sid (agent id, minted every request by the session middleware) and tp_user (authenticated user id, HMAC-signed, 30-day, HttpOnly, SameSite=Lax, Secure when base_url is https / TLS). W2-3 unifies them; for now only userIDFromCookie (exposed for that middleware) reads tp_user. OAuth CSRF state lives in a signed short-lived tp_oauth cookie (provider|state|target) — no server-side state.
