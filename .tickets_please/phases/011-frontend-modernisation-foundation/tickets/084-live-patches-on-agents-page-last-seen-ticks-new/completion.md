## Learnings
Live patches on the agents pages. New sse_patch_agent.go (renderAgentPatch moved out of the sse_render.go stub into its own file, mirroring sse_patch_phase.go/sse_patch_ticket.go).

THE KEY FAN-OUT TRICK: the /agents/{id} activity feed needs the agent's OWN ticket/comment events, but those events historically published only to ticket:/project:/phase: topics (the #081 learning explicitly flagged "extend Topics there" for #084). Rather than touch all 5 publish call sites, I added ONE line to svc.withActor — it already stamps ByAgentID, so appending eventbus.TopicAgent(agent.ID) to ev.Topics there makes EVERY actor-attributed event (create/move/complete/archive/comment) also fan out to agent:{actorID}. The agents-detail renderer only acts on created/completed/comment kinds; move/archive on agent:{id} are harmless no-ops. Single chokepoint, zero churn.

DELIVERY ISOLATION (why this doesn't leak): index subscribes ONLY global:agents (ticket events don't carry that topic → never seen there); detail subscribes agent:{thisID}+global:agents, and a ticket event by agent X carries agent:{X}, so it reaches a detail page only when X==thisID. The Bus delivers once PER matching subscribed topic, so detail gets the ticket event once (on agent:{id}) → one prepend, no dup. (global:agents on the detail page only ever carries AgentRegistered/AgentSeen, never ticket events.)

PATCH SHAPES: AgentSeen → inner-morph #agent-lastseen-{id} (shared selector on both the index <td> and the detail <dd> — one patch updates whichever page is open; content is the relative <time> via the new exported AgentLastSeen component). AgentRegistered → prepend #agents-tbody with AgentTableRow(row, highlight=true) (the .agent-row-new CSS one-shot flash). Ticket/comment → prepend #agent-activity-feed with the exported ActivityRow.

EXPORT FOR BYTE-IDENTICAL STREAMING: extracted the index row into exported templ AgentTableRow(a, highlight) + AgentLastSeen(a), and renamed activityRow→ActivityRow, so sse_patch_agent.go renders markup identical to a server render (same path the ticket-detail comment streaming uses via buildCommentRow/toTemplRow).
