Learnings:
- `html/template` doesn't fully exit the dep graph after the cleanup — templ-generated `_templ.go` files declare `template.HTML` parameters (and `internal/web/components/md/md.go` returns `template.HTML`), so `html/template` stays as a direct import via templ. Don't chase it out; the W1 acceptance criterion "drop from indirect graph" only holds for our live rendering code, not templ's typed-HTML primitives.
- The two embed FSes (`embeddedTemplates` + `embeddedStatic`) were independent — deleting one didn't touch the other. `dev.go` had only the `embeddedStatic` decl after W1 ports; `embeddedTemplates` was already gone in source, only `web_test.go` still referenced it. Pattern: when a test for a deleted symbol persists into cleanup, the production build passes but `go test` build fails with `undefined: <symbol>` — vet catches it before tests run, which is the fastest signal.
- Two stale TestMarkdown tests in `handlers_projects_test.go` were the only callers of `renderMarkdown`. Easiest fix was rewriting them to call `md.Render` (same goldmark config, same safety contract) rather than relocating into the `components/md` package. Kept the tests in `web` because they're conceptually testing the markdown contract from a handler's perspective and the seam doesn't change.
