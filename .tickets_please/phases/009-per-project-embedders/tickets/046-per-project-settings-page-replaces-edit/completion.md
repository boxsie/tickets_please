## Learnings
Sub-nav links live in partials/sidebar.tmpl, not _nav.tmpl as the ticket hint suggested. Always grep templates before editing — the ticket spec was off by one partial. The "Edited" page also had an action button that pointed to /edit on detail.tmpl — easy to miss without a comprehensive grep. The "<N> stale sidecars" status block is heavier than it sounds: walking the project tree to count mismatches each render is wasteful, and the worker queue depth is a leaky proxy. Skipping the count entirely (per the ticket's "simpler approach" hint) is the right call for v1; if users want it, add a cached count later. Service.UpdateProject already accepts EmbedProvider/EmbedModel as *string; passing them as nil leaves them untouched (the W1-T3 pointer-pattern paid off here). The legacy POST /p/slug back-compat route's old error path that re-rendered projects/edit had to fall through to renderer.Error since edit.tmpl is gone — minor cleanup.
