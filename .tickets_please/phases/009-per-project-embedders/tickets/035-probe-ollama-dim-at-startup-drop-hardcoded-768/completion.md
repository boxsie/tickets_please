## Work summary
Added `Probe(ctx) error` to `embed.Provider` interface. Both `*Ollama` and `*OpenAI` gained a `dim int` field, a `Probe` method that runs `Embed(ctx,"ping")` and stores `len(vec)`, and a `Dim()` that returns the probed value (panics with a clear message if Probe hasn't been called). Deleted `expectedEmbedDim = 768` from `internal/svc/service.go`; added a `Service.EmbedDim` field that's populated from the global Probe in `NewWithEmbed` (90s timeout). `internal/svc/hydrate.go` switched its dim check from the old constant to `s.EmbedDim`. Test fakes in `svc`, `worker`, `mcptools`, and `web` got a no-op `Probe` to satisfy the interface.

## Learnings
When you add a method to a widely-mocked interface like `embed.Provider`, every fake provider in every package needs the new method to compile — `internal/svc/fake_embed_test.go`, `internal/worker/fake_test.go`, `internal/mcptools/register_agent_test.go`, `internal/web/web_test.go`. Plan for that ripple. Also: a `Dim() int` that panics-before-Probe is a useful tripwire — caught a stale `TestNewFactory*` assertion on first test run rather than letting tests silently divide-by-zero. The "do NOT modify hydrate.go" file boundary in this ticket's brief had to be relaxed because deleting `expectedEmbedDim` from service.go forces a two-token swap in hydrate.go — when you delete a constant, the boundary goes wherever it's referenced.
