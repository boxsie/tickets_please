## Learnings
No existing helper for the config file path or Cfg.ConfigPath field — had to export config.UserConfigPath. The ticket hint to "look for an existing field on Cfg" was wrong relative to landed code; the path was previously private. Tests writing to ~/.tickets_please/config.yaml require a swappable path injection — added userConfigPathFn package-level var in web. SaveYAMLNode requires the file to exist before it can preserve comments; auto-create with a banner header on first save. Skipping the cfg-write mutex was a deliberate tradeoff: cfg writes happen only via this UI (rare), reads are loose per-field, and the chrome shows current values from cfg directly. Documented inline. Defaults only matter for new projects (per spec) — no live-mount provider rebuild needed; per-project Re-embed via the per-project Settings handles model migrations.
