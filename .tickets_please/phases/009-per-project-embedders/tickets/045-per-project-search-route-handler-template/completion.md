## Learnings
Both pre-flight worries about the search service layer turned out to be no-ops: SearchTickets/Comments/Learnings were already per-mount post-W2-T1, and MCP search_* tools were already correctly routed. When a phase ticket says "rework method X to use mount indexes", grep first — earlier tickets often did that work as a side effect. The mount.Embed.Embed(ctx,q) hint in the ticket was misleading: the handler never calls mount.Embed directly; embedding happens inside Service.SearchX, which is the cleaner layering. Don't pull that responsibility up into the web handler. CurrentSlug was already wired through PageOpts; just had to fix one error path (renderProjectFormError) that didn't populate it. The fakeEmbedder upgrade pattern is reusable: when web smoke tests need real-ish search, mirror svc.fakeEmbed's SHA256-derived vector function.
