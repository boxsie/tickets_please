Learnings:
CSS grid auto-placement gotcha: a row styled `grid-template-columns: auto 1fr auto` will silently break if you render MORE children than declared columns. Extra children flow into an *implicit second row*, and column 1 then auto-sizes to the WIDEST item across both rows — so a wide 4th child (here the attribution Chip added in #099) blows out column 1 and shoves the `1fr` title sideways. Whenever you add a child to a fixed-column grid row, either add a column or wrap the trailing items in a single cell. Fixed `.phase-wave-ticket` by wrapping badge+pill+chip in `.phase-wave-ticket-meta`. Same trap applies to any `.phase-wave-ticket`-shaped row.
