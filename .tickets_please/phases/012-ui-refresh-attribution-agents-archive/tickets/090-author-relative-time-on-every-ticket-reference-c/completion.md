## Learnings
Built the shared chip as its OWN package: internal/web/components/attribution (Chip(t *domain.Ticket) + CommentChip(c *domain.Comment) + Label(t) string). Critical placement decision: it imports ONLY domain + reltime, so the three page packages that need it (tickets, phases, projects) can all import it with zero cycle risk — putting it in the tickets package instead would have made phases→tickets a new cross-page dependency. The hint said "TicketAttribution(t)"; I named it attribution.Chip for a cleaner call site.

Author resolution (attribution.Label): CreatedBy.Name → CreatedFor.DisplayName (acting-for human) → "unknown". commentLabel: Author.Name → AuthorFor.DisplayName → "system". Old pre-attribution tickets correctly render "unknown" (verified live — the May-2 tickets show it, newer ones show "Claude Code").

Chip markup keeps each piece its own element (<span author><span sep>·</span><time>) so templ never glues author to time — sidesteps the reltime spacing trap entirely (no `{ " " }` needed when pieces are separate elements; the trap only bites when prose text sits inline-adjacent to an @component on the same line).

Surfaces wired: ticket_card.templ, phases wave_section.templ (the row both phases-index AND phase-detail AND the overview phase-lead share — one edit lit up all three), projects overview ready-list (full Chip) + activity/learnings (used the Label STRING helper inline — `{ attribution.Label(t) }{ " · updated " }` + @reltime.Time — to keep the existing updated/completed *semantic* time rather than the chip's created-time), ticket-detail Depends/Blocks lists + the metadata deps/parallel popovers, and search_results.templ all three tabs. Learning search hits had NO author on the svc.LearningHit — only CompletedAt — so I added CompletedAt to the projectspg.LearningHit mirror + handler and render time-only there (noted: author would need a svc change to carry the completing agent).

Testing approach: a few component-level unit tests in the attribution package (deterministic — set CreatedBy/CreatedFor and assert label fallbacks + chip structure) PLUS two cheap surface assertions on existing handler tests that already seed fixture-agent tickets (phases EnrichesWithWaves wave rows, and the ProjectSearch happy-path ticket hit) — assert the "ticket-attribution" class appears. Fixture-created tickets get CreatedBy populated from the registered test agent, so the chip renders with a real name in tests.
