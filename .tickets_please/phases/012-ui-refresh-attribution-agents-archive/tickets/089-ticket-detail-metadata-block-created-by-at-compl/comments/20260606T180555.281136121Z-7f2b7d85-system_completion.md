Key decisions / gotchas:
- DELIBERATE DEDUPE: Phase/Wave/Status/Archived-badge already live in the .ticket-meta header breadcrumb, so the metadata block does NOT repeat them (the ticket spec says "Phase: already in header — dedupe"). The block adds only what the header lacks: created/updated/completed attribution, acting-for, archived *timestamp*, deps/parallel popovers, entry key.
- ParallelizableWith was NOT resolved anywhere — DependsOn already had dependencyLinks() resolving ids→tickets for the existing Dependencies card. Extended that helper to a 3rd return value (depends, blocks, parallel) reusing its byID map, and added Parallel []*domain.Ticket to both ticketDetailData (web) and DetailProps (tickets pkg). The deps/parallel popovers are no-JS <details><summary>N</summary><ul>…</ul></details> — reused the same <a>+badge row markup as the full Dependencies card.
- reltime.Time spacing trap (again, see [[shared-reltime-helper]]): inside <dd> the agent name expr and the <time> must be on SEPARATE lines with an explicit `{ " " }` space — `{ agentName(t.CreatedBy) }{ " " }` then newline `@reltime.Time(...)` — or templ glues them ("Claude Code6d ago") or swallows the whole line as literal text.
- Responsive collapse = wrap the dl in `<details class="ticket-meta-block" open>` with a <summary>Details</summary>, then CSS `@media (min-width:48rem){ .ticket-meta-block-summary{display:none} }`. Since it's `open` by default, wide screens always show the grid and just hide the toggle; narrow screens get the disclosure. Clean, zero JS.
- Copy button: new static/copy.js, a single document-level delegated click listener on `[data-copy]` → navigator.clipboard.writeText + 1.2s "Copied" flash. Delegation means SSE-injected nodes work with no re-wiring. Had to add `<script src="/static/copy.js" defer>` to layout.templ (go:embed serves it automatically, but the tag isn't auto-injected). Degrades silently on insecure origins (the visible <code> is still selectable).
- Tested the component directly (render to a strings.Builder via TicketMetadata(p).Render(ctx, &sb)) rather than through the httptest server — deterministic, lets you set CreatedFor/Archived/CompletedBy/etc. precisely and assert field present-when-populated / absent-when-nil. The relative-time TEXT is nondeterministic (ShortNow uses time.Now), so assert on structure (<dt> labels, classes, data-copy, <time ) not the "6d ago" string.
