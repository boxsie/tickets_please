## Learnings
Adding a bulk mutation: reuse the single-item mutation rather than re-implementing the write/commit/event path. `ArchivePhase` snapshots candidate ticket IDs under `lp.Lock.RLock()` (resolving the phase via `resolvePhase(lp, idOrSlug)`), releases the lock, then calls existing `ArchiveTicket` per ticket — which takes the cache WRITE lock itself, so you MUST release the read lock before the loop or you deadlock (same pattern as `ApplyArchivePolicy`). Tolerate per-ticket errors (record as skipped, don't abort the sweep). Wiring checklist for a new MCP tool: register in `tools.go`, add handler, bump the group-count comment, AND add the name to `expectedTools` in tools_test.go + the count comment + SPEC.md's "**N tools**" line and the group table — the test asserts exact count (was 35→36; the in-code "31" comment was already stale). Phases aren't frozen, so adding follow-up tickets to a "done" phase works fine.
