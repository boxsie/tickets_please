## Learnings
The board's `boardHref` helper had two non-board callers — the ticket-detail and new-ticket breadcrumbs linked "back to the board". Killing the board means repointing those at /phases (renamed boardHref→phasesHref), not just deleting it; grep for the helper, not just the route. Same for the ticket-DELETE redirect (handleTicketDelete sent you to /p/{slug}/board on success). The sidebar already had a Phases link, so "Board link → Phases" really meant *remove* the Board entry (don't duplicate). ticket_card.templ/TicketCard survived: it was only rendered by board.templ, so it's now unreferenced, but it's the shared card primitive the W2 attribution wave (dc20c148) will reuse — left it in place (unused exported templ funcs don't fail the build). The redirect is 302 (StatusFound) per the ticket, kept forever as cheap insurance for stale links/agent memory. Test gotcha: the web test client is configured NOT to follow redirects (CheckRedirect→ErrUseLastResponse — same reason the delete test can assert 303+Location), so client.Get on /board observes the 302 directly. Three tests referenced /board and had to flip to /phases: the delete-happy redirect assert, the overview-detail link-presence loop, plus the two board-render tests I replaced with TestBoard_RedirectsToPhases.
