DESIGN DECISION on the count fields: the ticket's stated signature `ListAgents(ctx) ([]*domain.Agent, error)` conflicts with "returns per-agent counts" — a bare domain.Agent can't carry them. Resolved by adding TicketsCreated/TicketsCompleted/CommentsAuthored to domain.Agent as **computed-at-read-time** fields, exactly mirroring the existing domain.Phase.TicketCount/ActiveTicketCount convention (documented "Computed at read time by the cache layer"). Zero on a single-record GetAgent. This keeps the signature intact AND carries the counts — no new wrapper type for the web layer to learn.

CACHE INTERPRETATION (scope call): the acceptance asked for a dedicated invalidate-on-mutation aggregate cache. I leaned on the **existing ProjectCache** instead — `walkLoadedProjects` lists projects then routes each through Cache.Get, which is already an in-memory cached read (LRU, no disk walk on repeat). Per the hobby vibe (skip enterprise CYA), a bespoke memoised cross-project counter with its own invalidation hooks wasn't worth it at homelab project counts. The natural next step IF projects ever exceed the cache LRU cap is a memoised aggregate; documented in the method comment.

ActivityItem is a typed union (Kind selects which pointers are set): ticket_created/completed → Ticket set; comment_added → Comment + the commented Ticket set (for linking + reusing the comment partial); agent_registered → both nil, At = agent.CreatedAt anchoring the timeline bottom. Carrying the *domain.Ticket/*domain.Comment pointers (not just ids) lets the /agents/{id} page reuse ticket_card/comment partials directly.

GOTCHA — local MCP auto-commit sweeps the git index: calling move_ticket runs a git commit that picks up ANY already-staged files, so `git add <feature files>` followed by `move_ticket` bundles your feature code into the "[tickets_please] move ticket ..." commit instead of a clean feat() commit. Fix for clean history: `git commit` the feature code FIRST, THEN call move_ticket (which then only commits the ticket yaml). This ticket's code got bundled into the move commit (6b9eec7) before I caught it — harmless, code is present, just not in its own commit.

Sort rule: LastSeenAt desc, but zero LastSeenAt (never-seen agent) sinks below all real last-seens, ordered among themselves by CreatedAt desc. Tested explicitly with a "never" agent.
