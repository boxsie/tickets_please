## Learnings
The project overview and the /phases page must stay DIFFERENT surfaces: overview = dashboard (stats, status bar, ready/recent/learnings, compact phase summary), /phases = the collapsible per-wave drill-down. A prior change pasted the full `phases.PhaseList` (the wave dump) atop the overview — don't. Added `phases.PhaseSummary` (compact, non-collapsible, reuses `PhaseMeta`) for the overview and link out via a new `ui.CardProps{Href, LinkLabel}` header link. Test `TestProjectOverview_*` guards the boundary: it asserts the overview contains `phase-summary` + a /phases link but NOT `phase-wave-ticket`/`phase-row-summary` (ticket titles CAN still appear via Ready/Recent lists — don't assert against those). When changing the dashboard, update that test's expectations rather than reverting.
