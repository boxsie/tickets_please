## Learnings
Phase-less tickets are already in hand on handlePhasesIndex — it fetches ALL project tickets (limit 200) then bucketTicketsByPhaseAndWave drops PhaseID==nil. So surfacing them is just `bucketTicketsByWave(phaseLessTickets(tickets))` on that same slice; no extra ListTickets call needed. Reused the existing PhaseWaves + WaveSection components verbatim for the Unphased body — they're presentation-only and slug-threaded, so an "Unphased" pseudo-phase is purely a new summary row (UnphasedRow) wrapping the same wave body. Gave it id="unphased" so the deep-link ticket's ?phase=unphased anchor lands. CSS distinctiveness = dashed dimmer border + muted chevron + italic "No phase" label (no progress bar, since there's no single phase to measure). Inverted the old TestPhasesIndex_PhaseLessTicketsExcluded (which asserted orphans were HIDDEN) — that assertion is now exactly backwards, the orphan must appear but still not leak into a real phase row (structural </details>-chunk check). Two acceptance bullets are intentionally deferred to their owning W1 tickets: the overview lead-block reuse lands in overview-lead-with-phases (the block doesn't exist yet), and ?phase=unphased&wave=N server-side filter lands in deep-link-to-waves (it builds the ?wave=N focus filter machinery). UnphasedRow is exported precisely so the overview ticket can call it.
