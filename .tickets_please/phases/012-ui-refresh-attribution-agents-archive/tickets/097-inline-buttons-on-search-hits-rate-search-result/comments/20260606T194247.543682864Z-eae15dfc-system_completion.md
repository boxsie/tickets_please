Key facts:

- THE svc HITS ALREADY CARRY EntryKey. svc.TicketHit/CommentHit/LearningHit each have an EntryKey domain.EntryKey field (`<kind>:<id>`), so no derivation needed — just copy string(h.EntryKey) into the page mirror. Don't rebuild entry keys by hand.

- COUNTS ON INITIAL RENDER needed a new read-only svc method: FeedbackCounts(ctx, slug, keys) → map[EntryKey]FeedbackRecord, reading mount.Feedback.Get(key) per key (the store-level Get already returns Likes/Dislikes). No session required (display data); missing/never-rated keys are just absent (treated as zero). The handler gathers every hit's EntryKey, one FeedbackCounts call, threads Likes/Dislikes into the mirrors. Counts render only when >0 ("👍 3 · 👎 1").

- WIDGET = TWO REAL <form>s, not JS buttons. CSRF here is a `_csrf` FORM FIELD (checkCSRF reads r.Form.Get("_csrf")), NOT a header — so hx-post must carry it as a hidden input, which a real <form> does for free (and gives a no-JS full-POST fallback). 👍 = one-click submit (rating from the button's name=rating value=like). 👎 = a native <details> whose <summary> is the 👎 glyph; opening reveals the optional reason textarea + a dislike-submit form. The whole widget (span#hit-rating-...) is the hx-swap target outerHTML, so the POST response (the same RatingWidget rendered with Rated:true) replaces it in place → sticky "rated" state + ui.Toast thanks, surviving with zero client state.

- ENTRY-KEY DOM ID TRAP: entry keys contain a colon (ticket:uuid). A colon is legal in an HTML id but breaks a `#id` CSS/querySelector target (htmx resolves hx-target via querySelector). Sanitise: ratingWidgetID = "hit-rating-" + ReplaceAll(key, ":", "-"). Both the widget id and hx-target use the sanitised form.

- HANDLER maps RateOutput partial-success: out.Rejected (unknown/malformed key) → 422 with the per-key Error; bad/missing rating → 400; success → htmx renders the rated widget partial, no-JS gets flash+redirect to /search?q=. Route is RoleViewer (anyone who can see search can rate) — POST so it still goes through withCSRF via slugRole.

- ESC: native <details> can't close on Escape, so search-feedback.js is a 4-line delegated document keydown that strips [open] off any details.hit-dislike. Delegation → works on htmx-swapped widgets. Everything else (open/close, submit) is native, no JS.
