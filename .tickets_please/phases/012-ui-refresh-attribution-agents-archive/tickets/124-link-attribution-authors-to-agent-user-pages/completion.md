## Learnings
"No linking to the agents page" wasn't a missing route — `GET /agents` works fine (a stale May-31 binary was 404ing it during testing; rebuild the actual `./tickets_please` binary with `go build -o tickets_please ./cmd/tickets_please`, since `go build ./...` does NOT update it and the running server embeds static assets + routes at compile time). The real gap: `attribution.Chip`/`CommentChip` rendered the author as plain text even though `AgentLink`(→/agents/{id}) and `UserLink`(→/u/{id}) components + the `.attr-link` style already existed. Wired the chips to use them with the same fallback order as `Label` (CreatedBy.ID → CreatedFor → "unknown"), so names are unchanged but now clickable. When something "isn't linked," check whether the link component already exists and the call site just isn't using it.
