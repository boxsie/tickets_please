✅ Ticket completed.

Learnings:
Styling once-over of an ESTABLISHED design system ≠ greenfield bold redesign. The tickets_please UI already has a coherent dark dev-tool aesthetic with good tokens (`@theme` + legacy `--*` vars) and shared templ components; the highest-value move was fixing real defects (grid misalignment, blue-thumbs specificity, undefined `--bg-elev-1`) and adding small consistent affordances (Card header links, `.btn-warn`, `.archive-zone`), NOT a font/color teardown. Verify CSS in a REAL browser — google-chrome `--headless=new --screenshot`, then crop with ImageMagick `magick`; several bugs were invisible in source. Deliberately did NOT add global page-load animations: the UI uses datastar/htmx SSE morphing, and entrance animations would re-fire every time an element is morphed in. CSS source is `internal/web/static/_src/app.css` (tailwind v4); rebuild with `make css` (or `make build`), never hand-edit the generated `app.css`.
