Learnings:
The HTML details element is a clean combobox primitive when no JS framework is available — open/close is native, click-outside-close is one tiny script, no aria-expanded juggling. The summary cursor pointer plus list-style none reset is standard. The htmx HX-Current-URL header is the right way to recover the page URL inside a partial endpoint that gets called from a body-scoped event — without it, the partial only sees its own /_partials/sidebar URL. The sidebar refresh contract was set up to be slug-aware via a query param previously; HX-Current-URL is strictly better because it gives the full path the user is on, and the slugFromPath helper covers the small parsing detail. Putting aria-current page on the active per-project nav link satisfies both the existing test and accessibility good-practice. The dropdown layered above the rest of the sidebar via position absolute plus z-index 50 plus a heavy box-shadow gives it the right visual elevation; max-height plus overflow-y on the list keeps it bounded. Filter behaviour as data attributes (data-picker-item, data-name) keeps the inline script tiny — about 20 lines including click-outside-close. The dropdown closes when its details toggle event fires; resetting the input value plus restoring all items on close avoids a stale filter state on next open. The redundant in-page tabs on the project detail (Board / Phases / Waves / Summary) are now slightly redundant with the sidebar nav but I left them in scope-controlled — a future polish ticket can drop them.
