## Learnings
All four guards live in svc/tickets.go so MCP + web both inherit them. (1) CompleteTicket: idea check goes BEFORE the already-done check (an idea is never done anyway). (2) ListTickets ready_only: added `if t.Kind==KindIdea { continue }` INSIDE the ReadyOnly block (not the outer kind filter) so it holds even when a caller passes IncludeIdeas+ReadyOnly together. (3) MoveTicket: reject idea→non-todo (ideas stay parked in todo; comments still allowed). (4) depends_on/parallelizable_with idea-target rejection lives in TWO places — CreateTicket validates deps INLINE (its own loop, doesn't call validateTicketRefs), while UpdateTicket uses validateTicketRefs; I added the idea check to both. All error messages name promote_idea so the model self-corrects. Chose to hard-gate move (the ticket left it optional) — cleaner invariant: an idea is always in todo.
