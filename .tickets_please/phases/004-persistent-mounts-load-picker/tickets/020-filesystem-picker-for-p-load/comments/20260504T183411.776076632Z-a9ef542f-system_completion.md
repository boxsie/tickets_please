Learnings:
Browsers can't give the server an absolute filesystem path from input type=file (security feature), but a server-side directory browser is fine for localhost-only apps and produces a nicer UX than a freeform text input. The same handler can return both JSON and a rendered partial keyed off HX-Request — keeps the API and the htmx form in lockstep without two implementations. Hidden directories (starting with .) should be filtered from the LISTING but the marker check is still per-entry so a regular dir whose .tickets_please subdir was created later gets the badge. The form attribute on a button (form=load-form) lets a button living outside the form element submit it — useful for the picker layout where the Load this directory button lives in the breadcrumb area but the form wraps the picker so the hidden path input stays a sibling. Disabling a button via the disabled attribute is clearer than removing it because the user sees what's possible vs gated; pair it with a tooltip-style hint explaining the gate. The os emoji folder character (📁 vs 📂) is a low-effort visual cue for marker vs non-marker rows that doesn't require an SVG icon system. The handler does not redirect or chain through CSRF since it is GET-only; the actual mount is the existing POST /p/load which already has CSRF on it. Permission denied on a dir surfaces inline as friendly text rather than a 500 — important for cases where a user navigates into someone else's home dir.
