## Real learnings (truncated by MCP transport on complete_ticket)

- stringSliceFromAny already existed in the package (used by list_comments_scoped); reusing it for entry_keys saved hand-rolling array decode.

- The handler's defensive GetArguments → BindArguments pattern is now the right shape for ANY tool whose schema includes an array (or any non-scalar) — RequireString works on flat string params, but `req.GetArguments()["entry_keys"].([]any)` panics if Arguments came in as RawMessage rather than map[string]any. Worth porting this pattern to any other handler that touches arrays (currently: list_comments_scoped via stringSliceFromAny, the three search handlers via stringSliceFromAny for columns).

- Resolving the project via lp.Project.Slug AFTER Cache.Get (rather than from the input slug-or-id) is important — the mount registry is slug-keyed but Cache.Get accepts either UUID or slug, so a UUID input would route correctly through the cache but fail to find the mount if you fed it back into mountForSlug verbatim.

- The concurrent-rates test (20 goroutines + 1 trailing sequential → exactly 21 likes) proves the cross-handler call lock chain (handler → svc.RateSearchResult → mount.Feedback.RecordRating → store.WithProjectLock) doesn't lose updates.

- The rebuild+restart story for the user systemd unit is: `make build` (writes binary into the repo), `systemctl --user stop tickets_please.service`, cp into ~/.local/bin/tickets_please, start. Skip the stop and you get "Text file busy" because the kernel maps the running ELF.

- complete_ticket STILL rejects multi-paragraph (~1500+ char) fields over the HTTP streaming transport even with requireStringArgs in place, returning "required argument(s) not found: work_summary, learnings". The unit tests in mcptools/ (both map[string]any and json.RawMessage envelope) pass cleanly — so the bug is at a layer ABOVE requireStringArgs, likely in mcp-go's HTTP transport's argument-passing format. Worth its own follow-up ticket. Repro: just call complete_ticket through the HTTP MCP with field values that span multiple paragraphs.
