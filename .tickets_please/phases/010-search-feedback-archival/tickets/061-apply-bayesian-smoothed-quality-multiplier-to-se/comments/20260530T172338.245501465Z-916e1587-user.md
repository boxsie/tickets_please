## Real learnings (truncated by MCP transport on complete_ticket)

- The killer bug I hit first: applying a positive multiplier to negative cosine scores INVERTS desired ordering (multiplier × -0.06 = -0.054 is LESS negative than multiplier × -0.045 = -0.034, so the LOWER-cosine candidate sorts higher after rescore). Real-world ollama embeddings rarely produce negative cosine on a meaningful query, but the fakeEmbed test had wildly unrelated query and seed text → negative scores → broken test. Fix: clamp at 0 (pass anti-correlated hits through unchanged). This is a behavior, not a workaround.

- Second test fix less interesting: the indexed text is `Title + "\n\n" + Body` (see hydrateTicketBody). Queries for the body alone get a different vector than seeded. Make the query string EXACTLY match Title\n\nBody for fakeEmbed to give cosine ≈ 1.0.

- Per-mount rescore inside cross-mount searches is non-negotiable — each mount has its own QualityParams from its own project.yaml, and its own Feedback store. Doing one global rescore after merge would conflate configs from different projects. The cost: re-sort within each mount before the merge sort, but the merge sort is stable so the relative ordering within a mount survives.

- The raw_score propagation through the scored struct's new `raw` field is the cleanest way to thread the pre-multiplier value through the cross-mount paths. Trying to recover it via "rawScore = adjusted / multiplier" would round-trip-fail due to float32 precision.

- expandedRawLimit(limit) hard-capped at searchMaxLimit (50) — important the cap is unconditional rather than max(out, limit). clampSearchLimit already prevents inputs > 50 from reaching here in practice, but a contrived input shouldn't bypass the cap.

- The QualityParams kill switch (Enabled=false) is essential for A/B comparison: flip `feedback: {enabled: false}` in project.yaml, restart the mount (or wait for the next re-mount), and search reverts to pure cosine. Score == RawScore in that case so external tooling watching the delta sees zero adjustment.

- mountQualityParams(nil mount) returns the zero-value QualityParams (Enabled=false), so the registry-empty stdio fallback never applies the multiplier — exactly what we want for the bootstrap path that doesn't have a project.yaml.
