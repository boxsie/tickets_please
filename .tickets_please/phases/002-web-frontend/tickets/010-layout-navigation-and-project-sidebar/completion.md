## Learnings
HX-Request fall-through: render the page main block, not a separate partials/<n>.tmpl. Ticket 1 wired Page() to fall through to Partial(name, data) on HX-Request, intending each page to ship a sibling partials/<n>.tmpl. That's duplicated content for no benefit. Cleaner: parse pages/<n>.tmpl alone and ExecuteTemplate(w, main, data). Same source of truth, chrome stripped, full PageData still available. Added a third template cache (mains) alongside pages and partials. Ticket 3+ writers: if you want a different shape on HX-Request (e.g. return just a single ticket card after a move), call Partial(w, r, ticket_card, data) directly instead of relying on Page's fall-through.

ChromeProvider keeps the renderer test-friendly. Renderer needs sidebar projects + agent label + flash + CSRF token, all of which depend on Service + session. Threading those into NewRenderer would have made tests need a real Service for every render. Instead the renderer accepts a ChromeProvider interface (nil for tests), and app satisfies it via a Chrome(w, r) method back-referenced into NewRenderer at construction. Tests that need predictable chrome use a 5-line fakeChromeProvider; renderInline skips chrome entirely.

Tailwind v3 with semantic class names yielded 8 KB output, not the planned 30 KB. I went all-semantic (.topbar/.sidebar/.empty-state) rather than utility-first. Tailwind purges every utility it can't find in templates, so no utilities ship. Tailwind here functions as a PostCSS preprocessor with theme() colour resolution + a normalise/preflight + my own @layer components block. If a future ticket reaches for utilities, just use them — the next regen picks them up automatically. Documented in static/README.md.

Go's go:embed underscore-prefix exclusion is exactly what we want for sources. //go:embed static (recursive) automatically skips static/_src/. Keeps Tailwind sources out of the binary without an explicit ignore list. The underscore-prefixed templates (_layout, _nav, _sidebar) need to be listed explicitly in the embed directive — same exclusion rule, opposite direction. TestEmbeddedFS_HasExpectedAssets has both positive and negative assertions to lock this in.

Sidebar refresh contract for ticket 3. _sidebar.tmpl has hx-get=/p hx-trigger=sidebar-refresh from:body hx-swap=outerHTML hx-select=#sidebar. Ticket 3 (Projects CRUD) handlers that mutate the project set should respond with HX-Trigger: sidebar-refresh (response header). htmx fires the synthetic event on body, sidebar picks it up and re-fetches /p, hx-select pulls just the new aside out of the response, outerHTML replaces itself. GET /p in ticket 3 just needs to render the full page (which always includes the sidebar via _layout); hx-select handles plucking the right node. No special fragment endpoint needed.

Localhost banner uses r.Host, which reflects the request Host header — a reverse proxy can defeat it. Tested with curl -H Host: 192.168.1.5 and the banner appears even though TCP went to 127.0.0.1. That's intentional: informational warning, not a security boundary. README already calls this out.

CSRF token stability across renders. csrfToken() is pure HMAC over (purpose, agentID), so it's stable for the life of an agent's cookie (7 days). Forms submitted after a page idle for hours still validate. No need to rotate per request — the threat model is CSRF (cross-origin POST), which SameSite=Lax already mostly defeats; the token is belt-and-braces.

Eager-mount surprise during smoke testing. When I started the binary with no DATA_DIR override, it eager-mounted ./.tickets_please from cwd and GET / immediately 303 redirected to /p/tickets-please. Empty-state smoke needs DATA_DIR=$(mktemp -d). The serve subcommand has --data-root but no --data-dir flag — DATA_DIR env var is the only override. Worth knowing for ticket 8 (smoke tests).

Ticket 2 keeps handlers in package web (handlers_*.go) per the deviation established in ticket 1 — sub-packaging would import-cycle. Future tickets continue this pattern.
