## Learnings
html/template's loadPartial that ParseFS one file at a time silently breaks cross-partial template invocations — the second template lookup just returns no such template at runtime. The fix is to ParseFS partials/*.tmpl on every partial load (cost: parsing a couple dozen tiny files; benefit: any partial can {{template ...}} into a sibling without bespoke loaders). Ticket 5's reserved comments slot worked exactly as designed — replacing the placeholder section with a template call required zero handler refactoring beyond one new field on ticketDetailData. domain.Comment.FromColumn is *domain.Column not *string so the existing deref helper doesn't accept it; needed a separate derefCol helper since Column is a defined string type rather than a string alias. The htmx beforeend pattern with hx-on::after-request="if (event.detail.successful) this.reset()" is a clean way to clear the textarea on success while keeping typed text on a 422 — no JS framework needed. Service.CreateComment already trims whitespace and rejects empty bodies, so the handler just forwards the raw form value; the template's required minlength=1 is a UX hint, server enforcement is the source of truth. svc.MoveTicket emits a system_move comment automatically with FromColumn and ToColumn populated — those render as todo → in_progress in the meta line and let humans audit moves at a glance. Comments are immutable by design so the UI shows zero edit/delete affordances; tests assert on the absence of those classnames so future contributors get a fast signal if they accidentally add them.
