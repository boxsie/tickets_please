## Learnings
Ticket 8s scope is intentionally a long polish list but most of it was already shipped vertically by the earlier tickets — empty states for /p, /p/slug/phases, comment thread, and search results were all in place before this pass. The remaining gap was the boards per-column empty cell which the Go template range else block handles cleanly without a separate empty-state partial. A single linear smoke test (TestSmoke_EndToEnd) is more maintainable than a smoke suite of per-step tests because each step depends on the previous so splitting forces fixture duplication; keep them together until they are slow enough to need parallelisation. fakeEmbedder makes the search path testable without an Ollama process which is essential for CI; the trade-off is similarity scores are uninteresting so tests assert on response shape (200 plus search-hits or No matches) rather than on rank order. The /healthz route is mounted by main.go (cmd/tickets_please) not by web.Mount so freshServer (which only attaches web.Mount) returns 404 for it — the smoke tests asserting /healthz != 200 in the web-only fixture is actually a regression check that web routes do not shadow it. The audit-trail check (Author.Name non-empty plus contains web) is worth keeping as its own test because the synthetic agent identity is the load-bearing piece for cross-process attribution; if session.go ever drops the Web UI prefix the test fails fast and loud.
