## Learnings
**Sub-package would cycle.** The ticket-1 scope put handlers under `internal/web/handlers/` but `web.Mount` registering them creates `web → handlers → web` (handlers need Renderer/Deps). Resolved by keeping handlers in the same `web` package as `handlers_*.go` files. Future ticket-3+ should follow this pattern unless they want to move Mount itself into a `handlers` package.

**Cookie holds agent_id directly.** Original gotcha #1 said to cache cookie-uuid → agent-id via sync.Map[uuid]*sync.Once. Simpler: cookie value IS the agentID (HMAC-signed). On miss/expire, just call RegisterAgent again with a fresh random key (`web-ui:<8B hex>`) — no race guard needed because each mint generates a unique key, and `AgentStore.RegisterAgent` only returns `ErrAlreadyExists` for non-expired sessions with the same key. Dropped the sync.Map entirely.

**No AgentStore lookup-by-key.** `AgentStore` only exposes `ReadAgent(id)` + `WalkAgents(fn)`. Trying to keep a stable cookie-uuid that maps to an agent-id would require a WalkAgents per request. Embedding the agent-id in the cookie sidesteps this.

**Eager mount surprises empty-state tests.** `svc.NewWithEmbed` eager-mounts `cfg.DataDir` parent if it has `.tickets_please/project.yaml`. Running `serve` from the tickets_please repo means `ListProjects` is non-empty even with a fresh `--data-root`. Tests must set `cfg.DataDir = t.TempDir()` to exercise the empty-state branch; the production `--data-root` flag only overrides the central agent registry, not the project dir.

**Template wiring.** `{{define "_layout.tmpl"}}` in the layout file + `{{define "main"}}` in each page lets `template.New(...).ParseFS(layout, page).ExecuteTemplate(w, "_layout.tmpl", data)` work cleanly: the layout calls `{{template "main" .}}` and Go finds the page's definition in the same template set. Partials parse independently and execute by their own define-name (no layout drag).

**embed.FS placement.** `//go:embed` paths are relative to the .go file declaring the FS variable. Putting `embeddedTemplates` and `embeddedStatic` in `dev.go` (next to templates/ and static/) means the directives `templates/_layout.tmpl templates/pages/*.tmpl templates/partials/*.tmpl` and `static/*` resolve correctly.

**Go 1.22+ idiom for embed serving.** `http.FileServerFS(fs.Sub(embedded, "static"))` + `http.StripPrefix("/static/", ...)` works without any wrapper — no need for the older `http.FS(http.Dir(...))` dance.

**runtime.Caller(0) for dev FS.** The `--dev` mode uses `os.DirFS(filepath.Dir(runtime.Caller(0).File))` to find templates on disk relative to the package source. Stable as long as dev.go isn't moved within the package; documented in a comment so future restructures don't quietly break dev.

**Vendored htmx.** Pulled htmx@1.9.12 from `https://unpkg.com/htmx.org@1.9.12/dist/htmx.min.js` (48KB) plus the 0BSD `LICENSE` file from GitHub. Both committed to internal/web/static/. Future regen: same URL with a different version pin.

**Wiring is one line.** runServe didn't need restructure — just `web.Mount(mux, web.Deps{Service: s, Logger: log, Cfg: cfg, Dev: *dev})` after the existing `/mcp` + `/healthz` handlers, plus a `--dev` bool on the flagset and two extra fields on the startup log line. The `*svc.Service` is already concurrency-safe; no new locks needed for the web layer to share it with /mcp.

**Catch-all route gotcha.** `mux.Handle("/", ...)` matches everything not preempted by a more specific pattern, so the home handler must reject `r.URL.Path != "/"` with a 404 — otherwise /favicon.ico, /robots.txt, etc. would all render the home page with a 200. Future tickets adding `/p/`, `/tickets/`, `/search` will preempt those prefixes naturally.
