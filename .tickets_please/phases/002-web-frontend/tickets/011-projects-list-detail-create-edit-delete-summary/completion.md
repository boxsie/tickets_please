## Learnings
Sidebar refresh contract from ticket 2 (hx-get=/p hx-select=#sidebar) didn't actually work as documented. Page strips chrome on HX-Request, so /p with HX-Request returns only the main block with no sidebar; hx-select=#sidebar finds nothing. Fixed by introducing a dedicated /_partials/sidebar handler that returns just the aside fragment unconditionally, and pointing the sidebar hx-get there. handleSidebarPartial accepts an optional ?slug= so future tickets can preserve the active highlight across a refresh. Future tickets emitting HX-Trigger: sidebar-refresh just need to set the response header before WriteHeader; the sidebar handles the rest.

Go 1.22+ ServeMux pattern routing has a subtle gotcha: GET /p/{$} matches ONLY /p/ (with trailing slash) and ServeMux 301 redirects /p (no slash) to /p/. Tests with CheckRedirect=ErrUseLastResponse see the 301 instead of the page. Switched to plain GET /p (no slash, no {$}) for the index, which matches /p exactly and is the cleaner human-facing URL. The wildcard /p/{slug} still works because more-specific patterns take precedence.

Goldmark default config (no html.WithUnsafe) replaces raw HTML with the comment-style raw HTML omitted marker rather than HTML-escaping. Same safety guarantee — the dangerous tag never reaches the browser — but tests asserting HTML-escape sequences would fail. The right assertion is no script/iframe substring in output, not presence of escaped sequences. Test covers script + iframe + onerror= + javascript: as the canonical XSS surface.

Service.CreateProject enforces single-project-per-store at the host data dir level — calling it twice on the same store errors with ErrAlreadyExists: project X already exists at PATH (one project per data dir). The UI cannot serve POST /p as create-yet-another beyond the first; for additional projects, users must use POST /p/load to mount an external repo. UI surfaces the error inline if they try; documented in new.tmpl hint copy. Important context for tickets 4 and 5 since they do not have this single-of-its-kind constraint.

CSRF on POST is implemented as a withCSRF wrapper around handlers (router applies via wrap()). It calls r.ParseForm so handlers can read r.Form.Get without re-parsing. Token is the per-session HMAC; matches the form hidden _csrf field. Test scrapes the token from a freshly-rendered form page (GET /p/new) — that is also how a real browser does it.

In-place summary editor pattern: GET /p/slug/summary?edit=1 returns either the page (no HX-Request) or just the project_summary_edit partial (HX-Request true). POST /p/slug/summary returns either the project_summary_view partial (HX-Request) or 303s to /p/slug/summary (non-htmx). Both view and edit partials wrap themselves in div id=summary-region so the htmx outerHTML swap targets a stable id. The edit partial needs the CSRF token in its own data payload (since it is rendered via Partial, not Page — .Chrome.CSRF isn't reachable). Added a CSRF field to projectSummaryData; summaryCSRF(r) helper hoists the lookup.

renderer.loadPage now uses the glob partials/*.tmpl instead of enumerating each partial. Trade-off: every page load parses all partials whether they are used or not — fine for our scale (5-ish partial files), and avoids the bug where a new partial silently does not render until someone updates the load list.

Goldmark + GFM extension (tables, autolinks, strikethrough, task lists) is enough for project/phase/ticket summaries. No need for bluemonday or another sanitiser layer because the no-WithUnsafe config means raw HTML never round-trips. Add bluemonday only if a future feature needs to allow specific HTML tags (e.g. embedded SVG diagrams).

Delete confirmation: hidden confirm=yes form field plus a JS onsubmit=confirm(...) prompt rather than a custom modal. Belt-and-braces — a misconfigured form (no hidden field) gets refused server-side; a clicked-by-mistake button gets the browser confirm dialog. Future tickets adding destructive actions should mirror this pattern.
