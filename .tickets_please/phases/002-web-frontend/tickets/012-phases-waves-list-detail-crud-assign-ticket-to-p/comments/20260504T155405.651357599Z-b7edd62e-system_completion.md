Learnings:
Service.ListTickets returns three values (tickets, cursor, error), not two. Almost shipped a 2-var assignment that broke compile; the build error was the cleanest signal so don't outsmart the compiler. ListWaves with a nil phase pointer returns wave summaries over phase-less tickets only per its docstring — for a project-wide waves view across phases you must call ListTickets directly and bucket in-handler. Goroutine 1.22 method+path patterns let literal segments win over wildcards within the same prefix, so /phases/new and /phases/{phase}/edit coexist with /phases/{phase} without manual disambiguation. Test fixtures that need to drive Service through a session can RegisterAgent in the test then svc.WithSessionID a context to get past requireSession; this avoids round-tripping every fixture through the HTML form. The summary editor partial swap with #summary-region as the stable outerHTML target is reusable across resource types — the only differences are the URLs and the partial name; consider lifting into a generic helper if a third resource type wants the same UX (search summaries maybe).
