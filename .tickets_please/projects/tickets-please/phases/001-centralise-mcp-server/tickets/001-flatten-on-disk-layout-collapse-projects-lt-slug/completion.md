## Learnings
Single-project-per-Store ripples beyond paths: CreateProject uniqueness shifts from slug-equality to any-project-at-root, LockProject(slug) and LockGlobal collapse to one file, cache id-resolution only ever sees one record. DeleteProject must enumerate project-owned siblings instead of removing the whole slug subtree, since runtime files now live alongside. Legacy per-project lock is runtime not data so migrate skips and removes rather than colliding. Multi-project tests do not degrade to single-project shape gracefully — t.Skip with a forward-pointer is the honest fix. sed rewrites of comma-quoted Go literals are dangerous; run go vet after any bulk source rewrite. mcp-go session machinery and on-disk shape are independent — nothing in this ticket required touching mcp-go or the tools layer.
