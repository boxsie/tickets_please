Picking this up. Root-caused the misleading "required argument work_summary not found": mcp-go's RequireString reads args only via GetArguments(), which returns nil whenever Params.Arguments isn't a map[string]any (e.g. a json.RawMessage envelope) — then every field reads as "not found" even though present. Fixing with an encoding-robust requireStringArgs helper (BindArguments fallback + accurate multi-field error) and a regression test covering long/rich markdown.
