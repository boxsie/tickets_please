## Learnings
The Summary view at slash p slash slug slash summary is the LLM-loadable canonical doc — keep it untouched. The Overview at slash p slash slug serves humans and should be a state-of-play page. Once that distinction is clear they stop being redundant. ListTickets caps at 200 by default which is fine for a dashboard sample on typical projects; large projects get a representative slice and the board still shows the rest. Computing percentages with integer division and floor-rounding gives a sum less-or-equal-to-100 which is fine for visual segments. The metric-card hover-shape doesn't need interaction since the cards aren't clickable; the value plus label plus hint hierarchy is enough. The status-bar is a flex row of width-percent segments which is easier than a SVG chart and matches the board column dot colors exactly. learning excerpts as first-line-of-Learnings is a pragmatic preview; the user clicks through to the full ticket for the rest. The recent-activity feed is ticket-level not comment-level for v1 because comments require N+1 ListComments calls; a future service-level activity log could feed a richer per-event timeline. humanizeAgo is the kind of helper that belongs in a domain util package once a second caller appears; for now in handlers_projects is fine. The metric-card-success and metric-card-accent variants use a subtle linear-gradient from accent-tinted top to bg-elev which gives them visual weight without screaming.
